19 /// Shader parameter description
20 ///
21 /// This helper class is to be used by ShaderDesc to describe a shader's
22 /// parameters. The various properties of the parameter are usually
23 /// automatically filled from the corresponding Arnold parameter metadata in
24 /// init().
25 ///
26 /// When information is filled in, one can then generate the corresponding
27 /// Houdini UI template with getTemplate().
28 ///
29 /// Supported metadata:
30 ///
31 /// - @b houdini.label (STRING): The UI name of the parameter. Defaults to an
32 /// automatically captalized version of the Arnold parameter name.
33 ///
34 /// - @b houdini.help (STRING): The tooltip text.
35 ///
36 /// - @b houdini.type (STRING): A subtype for some parameters:
37 ///
38 /// * @b "file:image": Add an image file requester button for a string parameter.
39 ///
40 /// * @b "file:*.toto": Add a file requester with "*.toto" as file filter.
41 ///
42 /// * @b "menu:<type>:<language>": Add a menu to the parameter, of type
43 /// {replace|toggle|append|wild|exclusive|single}, whose items are
44 /// taken from the "houdini.menu" metadata (menu), or generated with a
45 /// Python (python) or HScript (hscript) script {menu|python|hscript}."
46 ///
47 /// * @b "opfilter[:<filter>]": The parameter is an operator path, with an
48 /// optional filter specification. For example, "opfilter:!!OBJ/CAMERA!!"
49 /// will filter out everything but cameras. See PRM/PRM_SpareData.h for a
50 /// complete list of valid filter values.
51 ///
52 /// - @b houdini.disable_when, @b houdini.hide_when (STRING): Parameter
53 /// expression to specify a condition under which a parameter is hidden or
54 /// disabled (grayed out). Hidding a parameter will not hide its corresponding
55 /// input. The expression syntax is described here:
56 /// http://www.sidefx.com/docs/houdini12.1/ref/windows/optype#conditionals
57 ///
58 /// - @b houdini.invisible (BOOL): Parameter will be created, but its UI will
59 /// not be visible.
60 ///
61 /// - @b houdini.no_menu_caps (BOOL): Do not auto-capitalize menu entries.
62 ///
63 /// - @b linkable (BOOL): This parameter cannot vary per sample and thus no
64 /// input should be created for it.
65 ///
66 /// - @b houdini.no_input (BOOL): Do not create a corresponding input for the
67 /// parameter. Use this metadata only in case you need to override the
68 /// "linkable" metadata.
69 ///
70 /// - @b houdini.skip (BOOL): Skip parameter altogether, it will not appear in
71 /// control nor inputs. @b Warning: This parameter will break compatibility
72 /// with old scenes containing the skipped parameter, causing Houdini to
73 /// crash at startup.
74 ///
75 /// - @b min, @b max, @b softmin, @b softmax (FLOAT): (hard, soft) x (minimum, maximum)
76 /// values for the parameter. Note that Houdini only supports a min and a max,
77 /// optionally qualified as hard or soft, so these values will be mapped to a
78 /// PRM_Range in the getRange() helper function.
79 ///
80 /// - @b houdini.menu (STRING): A user defined menu to be used for drop-down
81 /// menus, choice-lists, etc. The items are semi-colon separated token-label
82 /// pairs, eg. "one;Item number 1;two;Item number 2".
83 ///
84 /// - @b houdini.menu_script (STRING): An HScript or Python script to generate
85 /// token-label pair representing menu items.
86 /// Cf. http://www.sidefx.com/docs/houdini12.1/hom/assetscripts#parameter_menu_scripts
87 ///
88 /// - @b houdini.expression (STRING): A default expression to set on the
89 /// parameter. If prefixed with "python:", the expression will be treated as a
90 /// Python code, otherwise it will be treated as old expression script.
91 ///

159 /// Shader description
160 ///
161 /// This helper class stores all the information needed to build a Houdini node
162 /// UI. The data is gathered by automatically read the AtNodeEntry's metadata.
163 /// A template array can then be built on the heap with getTemplates(), suitable
164 /// for specifying a Houdini node UI.
165 ///
166 /// The following node metadata is supported:
167 ///
168 /// - @b houdini.label (STRING): The UI name of the shader node. Defaults to an
169 /// automatic captalized version of the Arnold node name.
170 ///
171 /// - @b houdini.icon (STRING): The icon name.
172 ///
173 /// - @b houdini.lod (STRING): The default LOD of freshly created nodes {LOW|MEDIUM|HIGH}
174 ///
175 /// - @b houdini.order, @b houdini.order[2-99] (STRING): A list of parameter
176 /// tokens, ordered as you wish then to appear in the UI. Parameters not
177 /// in the order list will be sorted per Arnold scanning order, and after
178 /// the order-specified ones. The list can be split in several successive
179 /// @b houdini.order<n> tags. This is because of a limitation of the number
180 /// of characters a string metadata can hold. As per Arnold 4.0.11.0, it is
181 /// only safe to keep string values below ~250 characters.
182 ///
183 /// - @b houdini.shader_type (STRING): Arnold shader category
184 /// {volume|light_filter|atmosphere}, default is a surface shader.
185 ///
186 /// - @b houdini.parm.<type>.<token> (STRING): Specify an additional
187 /// parameter, the <token> identifier can be used in the houdini.order
188 /// metadata to insert the parameter. The following types are supported:
189 ///
190 /// * @b houdini.parm.folder.<token> (STRING): A set of folders (tabs),
191 /// specify a semi-colon separated list of tab label and number of
192 /// parameters in tab pairs, eg. "Diffuse;4;Specular;5".
193 ///
194 /// * @b houdini.parm.heading.<token> (STRING): An underlined label, the
195 /// argument of the metadata is the label.
196 ///
197 /// - @b houdini.help_url (STRING): The help page URL.
198 ///
199 /// - @b houdini.category (STRING): The TAB submenu name under witch to file
200 /// the shader entry.
201 ///
