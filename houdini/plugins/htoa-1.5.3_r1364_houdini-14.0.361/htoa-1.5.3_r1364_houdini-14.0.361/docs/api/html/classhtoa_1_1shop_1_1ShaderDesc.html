Detailed Description

Shader description.

This helper class stores all the information needed to build a Houdini node UI. The data is gathered by automatically read the AtNodeEntry's metadata. A template array can then be built on the heap with getTemplates(), suitable for specifying a Houdini node UI.

The following node metadata is supported:

  • houdini.label (STRING): The UI name of the shader node. Defaults to an automatic captalized version of the Arnold node name.
  • houdini.icon (STRING): The icon name.
  • houdini.lod (STRING): The default LOD of freshly created nodes {LOW|MEDIUM|HIGH}
  • houdini.order, houdini.order[2-99] (STRING): A list of parameter tokens, ordered as you wish then to appear in the UI. Parameters not in the order list will be sorted per Arnold scanning order, and after the order-specified ones. The list can be split in several successive houdini.order<n> tags. This is because of a limitation of the number of characters a string metadata can hold. As per Arnold 4.0.11.0, it is only safe to keep string values below ~250 characters.
  • houdini.shader_type (STRING): Arnold shader category {volume|light_filter|atmosphere}, default is a surface shader.
  • houdini.parm.<type>.<token> (STRING): Specify an additional parameter, the <token> identifier can be used in the houdini.order metadata to insert the parameter. The following types are supported:
    • houdini.parm.folder.<token> (STRING): A set of folders (tabs), specify a semi-colon separated list of tab label and number of parameters in tab pairs, eg. "Diffuse;4;Specular;5".
    • houdini.parm.heading.<token> (STRING): An underlined label, the argument of the metadata is the label.
  • houdini.help_url (STRING): The help page URL.
  • houdini.category (STRING): The TAB submenu name under witch to file the shader entry.
