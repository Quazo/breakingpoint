Building

Supported Platforms

We support all Houdini 64-bit platforms, and try to support all compiler flavors. At the moment, this means:

  • Linux x86_64 gcc 4.4
  • Mac OS X 10.8
  • Windows 7 x64 vc9
  • Windows 7 x64 vc11

Requirements and dependencies

Because we rely on SOHO for the translation (see Geometry translation), HtoA will only run on Houdini FX or Houdini. Any Houdini Apprentice or Houdini Indie will not work unfortunately.

We generally support the last two production versions, but HtoA is reported to work with Houdini versions down to 12.5.

It is important to note that you need to run HtoA binaries that were compiled against the exact same version of Houdini you are running as Side Effects does not enforce strict ABI compatibility. Mismatched binaries may seem to work for a while if you are lucky, but will generally crash or fail more or less subtly.

All versions of Python that ship with Houdini are supported, at the moment this means Python 2.5, 2.6 and 2.7, depending on the platform.

For this documentation generation, you'll also need:

  • Doxygen 1.8.2
  • Graphviz 2.20

There are various external dependencies as well, you can get a list of them with "abuild -h", see below.

Building HtoA

To compile and install HtoA, a simple 'abuild' command is used. It's a wrapper script for SCons that will build and copy the files needed for the HtoA install:

./abuild install

A very useful option is "-j N" to spawn N parallel build processes:

./abuild -j 8 install

Type "./abuild -h" for the complete list of options, with a short description for each of them.

You can pass individual options on the command line:

./abuild MODE=debug install

However it's often easier to create a custom.py a the root of HtoA folder holding the bulk of your configuration:

# sample custom.py
MODE = 'opt'
WARN_LEVEL = 'warn-only'
SHOW_CMDS = True
COLOR_CMDS = False
API_DOCS_ENABLE = True
houdini_version = '13.0.509'
DOXYGEN = 'doxygen-1.8.2'
HOUDINI_ROOT = '/work/apps/houdini/dist/hfs%s' % houdini_version
ARNOLD_ROOT = '/work/apps/arnold/dist/Arnold-4.2.1.0'
TARGET_HOUDINI_PATH = 'dist/htoa_houdini-%s' % houdini_version
OPENVDB_ENABLE = True
OPENVDB_INCLUDE = '/my/openvdb/install/include'
OPENVDB_LIB = '/my/openvdb/install/lib'
TBB_INCLUDE = '/my/tbb/install/include'
TBB_LIB = '/my/tbb/install/lib'
ILMBASE_INCLUDE = '/my/ilmbase/install/include'
ILMBASE_LIB = '/my/ilmbase/install/lib'
ZLIB_INCLUDE = '/my/zlib/install/include'
ZLIB_LIB = '/my/zlib/install/lib'
BOOST_INCLUDE = '/my/boost/install/include'
JPEG_INCLUDE = '/my/jpeg/install/include'
JPEG_LIB = '/my/jpeg/install/lib'
TIFF_INCLUDE = '/my/tiff/install/include'
TIFF_LIB = '/my/tiff/install/lib'
ZMQ_INCLUDE = '/my/zmq/install/include'
ZMQ_LIB = '/my/zmq/install/lib'

Launching an Arnold-enabled Houdini

The easiest is to modify your houdini.env file and insert the path to the HtoA folder, see the Installation section of the main documentation.

An sample bash script (htoa.sh) is provided in the scripts folder to launch an Arnold-enabled Houdini. Modify it to suit your environment before launching.

If you want to customise the Houdini environment yourself, you will have to insert the HtoA directory in your HOUDINI_PATH environment variable before the factory Houdini folder. That's because for some of the Houdini configuration files, the first one in the path wins, such as $HH/soho/parameters/SOHOparameters.
