FAQ

HtoA is not working after editing the Arnold ROP with "Edit Operator Type Properties"

You have to edit CreateScript in arnold.otl, it has probably been overwritten when you saved it from the UI.

You need to expand the otl with the command:

hotl -X arnold_rop.contents arnold_rop.otl

edit arnold_rop.contents/CreateScript, the line to replace should be something like this:

# Create default properties
opproperty -f -F Properties $arg1 arnold3.3 default_output

then recompress with:

hotl -C arnold_rop.contents arnold_rop.otl

Cf. Indigo renderer integration thread by Jason Iversen at od[force].

How do I reload the arnold-4.0.ds dialog script without relaunching Houdini?

Open an HScript textport and type this command:

/ -> opproperty -R
Reloaded renderer properties

Note that the Reloaded renderer properties message does not mean the command was successful, watch the console from which you lanched Houdini for error messages.

How can I debug HtoA?

For Python code, if you're using Eclipse and the PyDev plugin, here's how you can set up a remote debugger server in Eclipse. This allows you to make breakpoints that will halt the execution of the Python interpreter that is embedded in Houdini and use the Eclipse debugger to step, visualize and even modify values on the fly.

For the C++ part of HtoA, you can use any debugger supported on your platform, such as GDB and attach to a running Houdini process. Many graphical front-ends exist for GDB, such as DDD or Eclipse.
